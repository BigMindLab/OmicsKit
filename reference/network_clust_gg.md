# Gene set network clustering with ggraph graphics

Builds a weighted gene set network from a Jaccard similarity matrix,
applies Louvain community detection, and returns up to four ggplot2
network visualizations using ggraph. Each plot is a standard ggplot2
object that can be further customized, saved with
[`ggplot2::ggsave()`](https://ggplot2.tidyverse.org/reference/ggsave.html),
or combined with patchwork.

## Usage

``` r
network_clust_gg(
  x,
  clust_result,
  jaccard_threshold = 0.5,
  min_degree = 2,
  superterms = FALSE,
  superterm_data = NULL,
  type = "all",
  seed = 174
)
```

## Arguments

- x:

  A `JaccardResult` object (output of
  [`geneset_similarity()`](https://danielgarbozo.github.io/OmicsKit/reference/geneset_similarity.md)).

- clust_result:

  A list returned by
  [`do_clust()`](https://danielgarbozo.github.io/OmicsKit/reference/do_clust.md),
  used to color nodes by hierarchical cluster assignment.

- jaccard_threshold:

  Numeric. Minimum Jaccard similarity required for an edge to be
  included in the network. Default: `0.50`.

- min_degree:

  Integer. Minimum node degree for a node to be retained in the network.
  Default: `2`.

- superterms:

  Logical. If `TRUE`, overlays super-term labels at community centroids
  on the relevant plots. Requires `superterm_data`. Default: `FALSE`.

- superterm_data:

  A list returned by
  [`get_network_communities()`](https://danielgarbozo.github.io/OmicsKit/reference/get_network_communities.md)
  (slot `$superterms`) or directly by
  [`get_superterm()`](https://danielgarbozo.github.io/OmicsKit/reference/get_superterm.md).
  Required when `superterms = TRUE`.

- type:

  Character. Which plot(s) to return. One of:

  - `"clean"` : colored nodes, no labels.

  - `"superterms"` : colored nodes with community super-term labels.

  - `"combined"` : colored nodes with super-term labels and individual
    node labels.

  - `"individual"` : colored nodes with individual node labels only.

  - `"all"` : all four plots. Default.

- seed:

  Integer. Random seed for the Fruchterman-Reingold layout and Louvain
  community detection. Default: `174`.

## Value

A named list. Depending on `type`, the list may contain any combination
of the ggplot2 elements `$clean`, `$superterms`, `$combined`, and
`$individual`. The list always contains:

- `$node_attributes`: A
  [`tibble::tibble()`](https://tibble.tidyverse.org/reference/tibble.html)
  with columns `NAME`, `cluster`, `community`, `degree`, `betweenness`,
  and `closeness` (plus `superterm` if `superterms = TRUE`).

- `$superterm_report`: A
  [`tibble::tibble()`](https://tibble.tidyverse.org/reference/tibble.html)
  with columns `community`, `superterm`, `n_genesets`, and
  `geneset_members`. Only present if `superterms = TRUE`.

## Details

Optionally overlays super-term community labels generated by
[`get_superterm()`](https://danielgarbozo.github.io/OmicsKit/reference/get_superterm.md).

For a base R igraph version that draws directly to the active graphics
device, see
[`network_clust()`](https://danielgarbozo.github.io/OmicsKit/reference/network_clust.md).

## Examples

``` r
if (FALSE) { # \dontrun{
# Requires igraph, ggraph, tidygraph
gsl <- list_gmts("path/to/gmt_folder/")
res <- read.csv("path/to/results.csv")

jac   <- geneset_similarity(gsl, res, fdr_th = 0.05)
clust <- do_clust(jac)
net   <- get_network_communities(jac, threshold = 0.3)

# Return all four plots as ggplot2 objects
plots <- network_clust_gg(
  jac,
  clust_result   = clust,
  superterms     = TRUE,
  superterm_data = net$superterms,
  type           = "all",
  seed           = 174
)

# Display individual plots
plots$clean        # no labels
plots$superterms   # community labels only
plots$combined     # community + node labels
plots$individual   # node labels only

# Save with ggsave (full ggplot2 objects)
ggplot2::ggsave("network_combined.pdf", plots$combined,
                width = 14, height = 14)

# Access node-level metrics
plots$node_attributes
plots$superterm_report

# Combine two plots with patchwork
library(patchwork)
plots$clean + plots$superterms
} # }
```
