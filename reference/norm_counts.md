# Normalized counts matrix for TCGA-LUAD

DESeq2 size-factor normalized counts derived from the TCGA-LUAD RNA-seq
dataset (16 tumor samples, 16 normal samples). Counts are divided by
DESeq2 size factors to correct for differences in library size across
samples, but remain in counts scale (not log-transformed).

## Usage

``` r
norm_counts
```

## Format

A numeric matrix with 21,330 rows (genes) and 32 columns (samples):

- rows:

  Ensembl gene IDs (e.g., `"ENSG00000141510"`).

- columns:

  Sample IDs matching the `patient_id` column of
  [sampledata](https://danielgarbozo.github.io/OmicsKit/reference/sampledata.md).

- values:

  Non-negative numeric. Size-factor normalized counts. Range: \[0,
  1,889,573\].

## Source

TCGA-LUAD STAR counts downloaded from the GDC Data Portal
(<https://gdc-hub.s3.us-east-1.amazonaws.com/download/TCGA-LUAD.star_counts.tsv.gz>).
Normalized with DESeq2::counts() (`normalized = TRUE`); generated by
`data-raw/deseq2_results.R`.

## Details

Suitable as input for
[`nice_VSB()`](https://danielgarbozo.github.io/OmicsKit/reference/nice_VSB.md),
[`detect_filter()`](https://danielgarbozo.github.io/OmicsKit/reference/detect_filter.md),
and
[`add_annotations()`](https://danielgarbozo.github.io/OmicsKit/reference/add_annotations.md).
For dimensionality reduction methods
([`nice_PCA()`](https://danielgarbozo.github.io/OmicsKit/reference/nice_PCA.md),
[`nice_UMAP()`](https://danielgarbozo.github.io/OmicsKit/reference/nice_UMAP.md),
[`nice_tSNE()`](https://danielgarbozo.github.io/OmicsKit/reference/nice_tSNE.md))
use
[vst_counts](https://danielgarbozo.github.io/OmicsKit/reference/vst_counts.md)
instead, which removes the mean-variance dependence of RNA-seq data.
